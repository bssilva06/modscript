1. Devvit Technical Overview: Core ComponentsA Devvit Web app is split into three main areas:Client (/src/client/): Where your HTML, CSS, and JavaScript live. This renders in a webview within the Reddit post.Server (/src/server/): A Node.js environment (supporting frameworks like Hono or Express) that handles API requests from your client and interacts with Reddit's Redis and Reddit API.Configuration (devvit.json): Defines the app's metadata, permissions, and settings.Key Capabilities & AdvantagesStandard Tech Stack: You can use industry-standard tools like React, Phaser, or Three.js.Built-in Hosting: Reddit continues to host the services for you.AI Friendly: Because it uses standard web patterns, your code is more easily interpreted and assisted by AI development tools.Important LimitationsLimitationImpactServerless EndpointsServers run only long enough to return a response; no long-running streams.Package RestrictionsYou cannot use fs or native packages; external services (like OpenAI) must be used instead.Network LockThe client can only communicate with the app's own webview domain (all other external calls must go through your backend).Ephemeral StoragelocalStorage clears whenever you update the app; you should use Redis for persistent data.Technical SpecsMax Request Time: 30 seconds.Max Payload Size: 4MB.Max Response Size: 10MB.Endpoint Prefix: All server endpoints must start with /api/.

Essential "Must-Haves"Every devvit.json file requires two primary things:name: A 3-16 character unique slug for your app (lowercase letters, numbers, and hyphens only).Component Definition: You must include at least a post object (for Webview apps) or a server object (for Node.js background apps).2. Note: It only supports CommonJS (cjs), not ES modules.permissionsThe security layer. You must explicitly enable plugins like http (and list allowed domains), redis for storage, and reddit for API access.triggersMaps Reddit events (like onPostCreate or onCommentSubmit) to specific internal API endpoints in your code.3. UI & Interaction Settingsmenu: Allows you to add custom buttons to the Reddit interface. You can define if the button appears for "moderators" or "users" and where it lives (post, comment, or subreddit).scheduler: Used for automation. You can set up "cron" jobs (e.g., 0 2 * * * for a 2:00 AM daily cleanup) to run specific tasks automatically.marketingAssets: Where you link your app's required 1024x1024 PNG icon.4. Developer Toolsscripts: You can define build commands (like vite build) that the Devvit CLI will run when you use devvit playtest or devvit upload.dev: Lets you hardcode a default subreddit for testing so you don't have to type it every time you launch your app.Best Practice TipThe documentation strongly recommends including the $schema property at the top of the file. It highlights how Reddit provides a free, scalable infrastructure to handle logic away from the client. 1. Data & Storage Redis: A free key-value database for persistent app data, limited to the specific installation scope. Secrets Storage: Securely stores API keys or sensitive credentials provided by moderators. Cache Helper: Improves performance and reduces external API costs by caching server-side fetch requests. 2. Automation & Events Triggers: Enables automated actions tied to Reddit events, such as a new post being created or a comment being submitted. Scheduler: Allows you to run "cron-like" tasks at specific times (e.g., daily cleanup or hourly status updates). 3. External & Content Interaction HTTP Fetch: Connects your app to external servers (like AI models or third-party databases), provided the domains are reviewed. Media Uploads: Lets users upload images directly to Reddit’s CDN while maintaining built-in safety checks. Integration & Fallbacks Reddit API: The standard gateway to query information like upvotes, comments, and post details. Text Fallback: Ensures your interactive app still displays meaningful text content on platforms that don't support Devvit (like Old Reddit).

7. This HTTP Fetch page explains how your app can communicate with the outside world. It allows you to pull data from external APIs or send information to your own servers.1. Enabling AccessBy default, all external network requests are blocked for security. You must explicitly "allow-list" domains in your devvit.json file:The Config: You add an http permission block with an array of specific domains.The Rules: You cannot use wildcards (no *.google.com). You must provide the exact hostname (e.g., api.openai.com).Approval: When you upload your app, Reddit admins review these domains. If approved, they are allow-listed for your app only.2. Server-Side vs. Client-SideHow you use fetch depends entirely on where the code is running:ContextCapabilitiesRestrictionsServer-side (/server)Can call any approved external domain.Must use https. 30-second timeout.Client-side (/client)Cannot call external domains.Can only fetch your app's own /api endpoints.3. Key Limitations & RequirementsHTTPS Only: Standard http requests are not supported; everything must be encrypted.Methods: Supports standard REST actions: GET, POST, PUT, DELETE, OPTIONS, and PATCH.Legal Requirements: If your app uses fetch, you must provide a Privacy Policy and Terms of Service in your app details.Global Allow-list: Some domains (like discord.com, googleapis.com, and openai.com) are pre-approved, but you still have to list them in your config to use them.4. Troubleshooting TimeoutsIf you hit a "context deadline exceeded" error, it means your request took longer than 30 seconds. The docs recommend:Moving the request to a background task using the Scheduler.Optimizing your external server's latency. This HTTP Fetch Policy page details the specific rules and approval process for connecting your app to external domains. Since you're working with AI and developer tools, understanding these "allow-list" tiers is critical for your project's architecture. 1. The Three Approval Categories Reddit categorizes every domain request into one of these buckets: Public APIs: High-reputation services like api.openai.com or api.wikipedia.org. These are generally approved if they have public documentation and a valid use case. Limited Cloud Providers: Specific subdomains for databases or hosting (e.g., your-app.supabase.com or my-bucket.s3.amazonaws.com). To get these approved, you must prove that Devvit’s internal tools (like Redis) can't handle your specific needs (e.g., needing a relational database for complex queries). Personal Domains: Standard personal websites (e.g., myportfolio.com) are almost always rejected. If you need to use one, you must provide a heavy justification for why Devvit's server cannot support the task. 2. Strict Formatting Rules When you add domains to your devvit.json, they must follow a rigid format or the build will fail: Exact Hostnames Only: Use nytimes.com, not https://nytimes.com. No Wildcards: You cannot use *.example.com. You must specify api.example.com. No Paths: Use example.com, not example.com/api/v1. 3. Compliance & Documentation If your app uses external fetching, you have higher "paperwork" requirements: README Requirements: You must include a "Fetch Domains" section in your README that lists every domain and explains exactly why it is necessary for the app to function. Legal Links: You are required to provide a Privacy Policy and Terms of Service link in your app configuration. 4. Global Allow-list Some domains are "globally allowed," meaning they are pre-vetted by Reddit. However, you must still list them in your app's config to use them. These include: AI/ML: api.openai.com, generativelanguage.googleapis.com. Social/Tools: discord.com, slack.com, youtube.googleapis.com. Data: finance.yahoo.com, wikipedia.org, api.weather.gov.

Settings and Secrets: This Settings and Secrets page explains how to manage configuration data and sensitive credentials for your app. It distinguishes between what a moderator can change and what you, the developer, must keep secure. 1. Two Scopes of Configuration The devvit.json file uses two distinct scopes to handle data: Subreddit Settings: These are installation-specific. They allow moderators to customize the app for their own community (e.g., a "Welcome Message" or "Auto-moderation" toggle). Global Settings & Secrets: These are set by you, the developer. They are shared across every single installation of the app and are perfect for things like API base URLs or environment toggles. 2. Managing Secrets Secrets are a specialized type of global setting (isSecret: true). Security: They are encrypted and never exposed in the UI. CLI Managed: Unlike regular settings, secret values can only be set by the developer via the CLI using npx devvit settings set <key>. Requirement: You must have at least one active installation (even a local playtest) before you can store a secret value. 3. Supported Data Types You can define settings using the following input types: String / Number / Boolean: Standard text, numeric, or toggle inputs. Select / MultiSelect: Dropdown menus for single or multiple choices. 4. Input Validation To prevent moderators from entering invalid data, you can point a setting to a Validation Endpoint: You define a validationEndpoint in your JSON. Your server implements a handler that checks the value. If the value is invalid (e.g., a negative number for "Account Age"), you return an error message that the moderator sees in their settings dashboard. 5. Accessing Data in Code In your backend (server/index.ts), you use the settings object to retrieve these values. Since these calls are asynchronous, it is a best practice to use Promise.all if you need multiple settings at once: TypeScript import { settings } from '@devvit/web/server'; const apiKey = await settings.get('openaiApiKey'); const model = await settings.get('aiModel'); Technical Limitations Size Limit: Each setting value is capped at 2KB. Secrets Scope: Secrets must be global; they cannot be subreddit-specific. Environment Variables: Note that .env files only work during local playtesting; for production, you must use the Secrets storage.

Data Privacy BoundariesDevvit strictly protects certain user data. Your app cannot access:A user's subscribed subreddits.Their voting history (upvoted/downvoted content).Their saved content or browsing history.Private profile information or friend lists.4. Common ActionsThe page provides code templates for the most frequent tasks:Submitting a Custom Post: Used to create the "interactive" posts your app lives in.Submitting a Comment: Useful for automated mod responses or "mopping" feedback.Note: The documentation advises against "low-signal" comments (like level-up pings) to keep the comment section high-quality.5. High-Value Methods The page lists dozens of methods, but these are the ones most relevant to the projects you've been exploring: Content Management: submitPost() and submitComment(): Create new content. Navigation & Discovery The client also includes specialized "Accessors" and "Listings": modMail: A dedicated getter to interact specifically with ModMail conversations. Listings: Methods like getHotPosts() or getNewPosts() return a Listing object, which allows you to iterate through pages of content rather than just getting a single snapshot. 4. Technical Note for Developers Permissions: Just because a method exists in the documentation doesn't mean it will work immediately. Your app's devvit.json must have the correct permissions, and for sensitive actions (like banUser), the person running the app must have those moderator rights in the subreddit. Asynchronous Nature: Almost every method here returns a Promise, meaning you'll be using await constantly in your logic.

 10. WikiPage class: This WikiPage documentation defines the class used to read, edit, and manage subreddit wiki pages. If you're building a tool that needs to store community guidelines, FAQs, or resource lists directly on Reddit, this is the interface you'll use. 1. Data Accessors (What you can "Read") The class provides direct access to the metadata and content of a specific wiki page: content & contentHtml: Retrieve the page body in either raw Markdown or rendered HTML. name: The unique identifier/URL slug of the page (e.g., "index" or "config"). revisionAuthor & revisionDate: Identify who last edited the page and when. revisionReason: Access the brief summary provided by the editor for that specific change. 2. Content Management Methods These are the primary "Write" actions available for the wiki: update(content, reason?): The most common method. It saves new content to the page. You can optionally provide a reason which appears in the page's revision history. revertTo(revisionId): Allows your app to undo changes by rolling the page back to a specific previous version—useful for automated "vandalism" protection. getRevisions(): Fetches a Listing of previous versions, which you can use to track changes over time. 3. Permissions & Settings Beyond the content itself, you can manage who can interact with the page: addEditor(username) / removeEditor(username): Programmatically grant or revoke a specific user's ability to edit this wiki page. getSettings() / updateSettings(): Access the WikiPageSettings to control visibility (e.g., is the page public or mod-only?) and editing permissions (e.g., required karma or account age). Technical Integration To get an instance of this class, you typically use the context.reddit client: TypeScript const wikiPage = await context.reddit.getWikiPage("subredditName", "pageName"); await wikiPage.update("New wiki content here", "Updated via Devvit app");
